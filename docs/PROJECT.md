# Проектная документация

## 1. Цель

Разработать многопользовательское веб-приложение для персонального планирования. После входа каждый пользователь работает только со своими задачами.

## 2. Функциональные требования

1. Регистрация, вход и выход из аккаунта.
2. Создание задачи с названием, описанием, приоритетом и сроком.
3. Просмотр личного списка и статистики.
4. Изменение, завершение и удаление только собственных задач.
5. Поиск и фильтры.
6. Постоянное хранение данных.
7. Адаптивный интерфейс и тёмная тема.

## 3. Технологии

- Python 3.10+ и Flask;
- SQLite;
- Jinja2, HTML5, CSS3 и JavaScript;
- Werkzeug для хеширования паролей;
- pytest для тестирования.

## 4. Архитектура

`auth.py` отвечает за аккаунты и сессии, `routes.py` принимает запросы задач, `models.py` реализует правила и запросы с проверкой владельца, `database.py` управляет SQLite и миграцией, шаблоны отвечают за интерфейс. Фабрика `create_app` упрощает настройку и тестирование.

## 5. База данных

Таблица `users`:

| Поле | Тип | Назначение |
|---|---|---|
| id | INTEGER, PK | идентификатор пользователя |
| username | TEXT, UNIQUE | имя для входа |
| password_hash | TEXT | необратимый хеш пароля |
| created_at | TEXT | время регистрации |

Таблица `tasks`:

| Поле | Тип | Назначение |
|---|---|---|
| id | INTEGER, PK | идентификатор задачи |
| title | TEXT, NOT NULL | название, 1–120 символов |
| description | TEXT | описание до 1000 символов |
| priority | TEXT | low / medium / high |
| due_date | TEXT | дата ISO 8601 |
| is_completed | INTEGER | 0 или 1 |
| created_at | TEXT | время создания |
| updated_at | TEXT | время изменения |
| user_id | INTEGER, FK | владелец задачи |

Связь `users 1 → N tasks` использует каскадное удаление. Индексы созданы для владельца, статуса, срока и приоритета. Схема и миграция старой версии применяются автоматически.

## 6. Маршруты

| Метод и путь | Назначение |
|---|---|
| GET/POST `/auth/register` | регистрация |
| GET/POST `/auth/login` | вход |
| POST `/auth/logout` | выход |
| GET `/` | личный список, поиск, фильтры и статистика |
| POST `/tasks` | создать задачу |
| GET/POST `/tasks/<id>/edit` | изменить свою задачу |
| POST `/tasks/<id>/toggle` | переключить статус |
| POST `/tasks/<id>/delete` | удалить свою задачу |

## 7. Безопасность и тесты

Пароли хешируются `generate_password_hash`. Сессия содержит только идентификатор пользователя. Получение и изменение задач всегда выполняется одновременно по `id` и `user_id`, поэтому прямой адрес не открывает чужие записи. Сервер проверяет поля, SQL-запросы используют параметры.

Восемь тестов проверяют авторизацию, регистрацию, валидацию, полный жизненный цикл задачи, 404 и изоляцию двух пользователей. Результат: **8 passed**.
